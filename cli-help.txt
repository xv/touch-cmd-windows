SYNTAX
    touch [OPTION]... FILE...

DESCRIPTION
    Updates the access and modification timestamps of each file specified by the
    FILE argument to the current time of day.

    The -a and -m flags are set by default.

OPTIONS
    -C          Change only the creation timestamp. The last access and last
                write timestamps are not affected unless -a and -m flags are
                also set.

    -a          Change only the last access timestamp. The creation and last
                access timestamps are not affected unless -C and -m flags are
                also set.

    -m          Change only the last write timestamp. The creation and last
                write timestamps are not affected unless -C and -a flags are
                also set.

    -c          Do not create FILE if it does not exist.

    -d          Do not dereference symbolic links. If FILE is a symbolic link,
                its timestamp will be changed rather than that of the file it
                refers to.

    -A OFFSET   Adjust the timestamps of FILE by OFFSET, which must be in the
                format "[-][[hh]mm]ss". The parts of the argument represent the
                following:

                    -    Make the adjustment negative, moving time backward.
                    hh   Hours (00-99).
                    mm   Minutes (00-59).
                    ss   Seconds (00-59).

                If FILE does not exist, it will be created with adjusted 
                timestamps, unless the -c option is specified.

    -r REFFILE  Use the timestamp of the file specified by the REFFILE argument
                instead of the current time of day. This option cannot be
                combined with -t.

                If -A is specified, the adjustment will be applied to the
                referenced timestamp.

    -t STAMP    Use the timestamp specified by the STAMP argument, which must be
                in one of the following ISO 8601 basic or extended formats:

                    Calendar date format
                        YYYYMMDD[Thh[mm[ss[.SSS]]][Z|±hh[mm]]]
                        YYYY-MM-DD[Thh:mm[:ss[.SSS]][Z|±hh[:mm]]]

                    Ordinal date format
                        YYYYDDD[Thh[mm[ss[.SSS]]][Z|±hh[mm]]]
                        YYYY-DDD[Thh:mm[:ss[.SSS]][Z|±hh[:mm]]]

                    Week date format
                        YYYYWwwD[Thh[mm[ss[.SSS]]][Z|±hh[mm]]]
                        YYYY-Www-D[Thh:mm[:ss[.SSS]][Z|±hh[:mm]]]

                The parts of the argument represent the following:

                    YYYY    Year (1601-9999).
                    MM      Month (01 to 12).
                    D       Day of week (1-7, 1=Monday, 7=Sunday).
                    DD      Day (01-31).
                    DDD     Day of year (001-366).
                    ww      Week of year (01-53).
                    T       Separates date and time components.
                    hh      Hour of day (00-23).
                    mm      Minute of hour (00-59).
                    ss      Second of minute (00-59).
                    SSS     Millisecond of second (000-999).
                    Z       Indicates timestamp is in UTC.

                Date components are validated against calendar rules for the
                specified year. This means "DD" must be valid for the given
                month and year (e.g., Feb. 29 is only accepted in leap years);
                "DDD" must not exceed 365 in non-leap years; and "ww" must not
                exceed the number of ISO weeks in the specified year.

                If the time is omitted, midnight local time is assumed.

    -h          Display this help information and exit.

    -v          Display version information and exit.
